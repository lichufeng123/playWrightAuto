# Playwright 切环境与切账号

## 默认行为

- 不设置 `PW_ENV` 时，默认跑 `test`
- 不设置 `PW_USER` 时，会按环境自动选默认账号
- `test` 默认账号是 `testUser`
- `prod` 默认账号是 `prodUser`

## 推荐写法

### 1. 官方 `npx playwright` 写法

测试环境 UI：

```powershell
$env:PW_ENV='test'
$env:PW_USER='testUser'
npx playwright test --ui
```

正式环境 UI：

```powershell
$env:PW_ENV='prod'
$env:PW_USER='prodUser'
npx playwright test --ui
```

低余额账号：

```powershell
$env:PW_ENV='test'
$env:PW_USER='lowBalanceUser'
npx playwright test --ui
```

### 2. 项目封装写法

测试环境 UI：

```powershell
node scripts/pw-run.js --env test --user testUser -- --ui
```

正式环境 UI：

```powershell
node scripts/pw-run.js --env prod --user prodUser -- --ui
```

低余额账号：

```powershell
node scripts/pw-run.js --env test --user lowBalanceUser -- --ui
```

## 临时切自定义账号

如果不用预置账号，直接指定手机号和验证码：

```powershell
$env:PW_ENV='test'
$env:LOGIN_PHONE='13246034775'
$env:LOGIN_CODE='1234'
npx playwright test --ui
```

要求：

- `LOGIN_PHONE` 和 `LOGIN_CODE` 必须同时设置
- 设置了 `LOGIN_PHONE`/`LOGIN_CODE` 后，会优先使用这组账号，不再读 `PW_USER`

## 强制刷新登录态

登录态过期或想重新登录时：

```powershell
$env:PW_REFRESH_STATE='1'
npx playwright test --ui
```

或者：

```powershell
node scripts/pw-run.js --env test --user testUser --refresh -- --ui
```

## 常用组合

工作流失败返还用例：

```powershell
node scripts/pw-run.js --env test -- tests/smoke/workflow_failure.spec.ts -g "异常场景：任务失败后返还赛点" --project=chromium --workers 1 --reporter=line
```

低余额专项：

```powershell
node scripts/pw-run.js --env test --user lowBalanceUser -- tests/smoke/workflow_low_balance.spec.ts --project=chromium --workers 1 --reporter=line
```
