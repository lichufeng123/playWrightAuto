

## 总体结论

本次测试用例修订基于 Review 报告对原始 67 条用例进行了全面审查与更新。修订后用例总数为 75 条（原 67 条修订 + 新增 8 条）。主要修订内容如下：

1.  **严重问题修正**：针对 Review 指出的与 PRD 矛盾点（如重新生成确认弹窗、赛点返还逻辑），已根据审核意见修正预期结果。
2.  **覆盖遗漏补充**：针对宫格参数、视频格式约束、模式切换、撤销边界等遗漏场景，新增 8 条专用测试用例（REVIEW-001 至 REVIEW-008）。
3.  **预期结果细化**：针对模糊不可执行的用例（如脚本逻辑、模式提示），已细化验收标准；针对 UI 细节（如颜色值、像素值），已在备注中标注来源或确认状态。
4.  **重复用例保留**：针对 Review 建议合并的重复用例，因审核意见明确为“不同意合并”（分属不同模块或场景），予以保留。
5.  **待确认项标注**：针对 PRD 未明确或存在矛盾的功能点（如单元格转节点后内容是否清空、导出格式），已在用例备注中保留问题说明，标记为需产品确认。

## 按审核意见处理情况

本次修订严格按照 Review 报告中的审核意见进行处理，分类统计如下：

1.  **已落实建议**：
    *   所有审核意见为“同意”、“确实有”、“没问题”、“保留”的项，均已修正用例预期结果或补充备注说明（如 TP-003, TP-017, TP-057, 覆盖遗漏 1/2/3/4/6 等）。
    *   针对 PRD 不一致但确认有依据的项（如 2000 字限制、UI 像素值），已保留用例并在备注中注明依据来源。

2.  **忽略建议**：
    *   **模型调用规则测试**：Review 建议补充各模式默认模型验证，审核意见为“这种不需要 测不到”，故未新增相关用例。
    *   **用例合并建议**：Review 建议合并 4 组重复用例，审核意见均为“不同意”，故保留原用例结构。
    *   **自动化建议**：Review 提出的自动化优先级建议，审核意见为“待人工审核”，本次修订不涉及自动化脚本编写，仅保留用例。

3.  **暂定/备注保留**：
    *   针对审核意见为“暂定”或“需产品确认”的项，未强行修改预期结果，而是在用例备注中保留了问题说明，以便后续跟踪。
    *   涉及项：TP-024（缩减弹窗）、TP-033~035（颜色/样式值）、TP-039（导出格式）、TP-042（转节点后单元格内容）、TP-055（转节点后单元格内容）、REVIEW-008（缩减弹窗确认）。

## 仍需人工确认

以下功能点因 PRD 描述不明确或存在矛盾，虽已写入用例但预期结果仍需产品或设计最终确认，测试执行前需重点跟进：

1.  **单元格转节点后原内容是否清空**：
    *   涉及用例：TP-042, TP-055, TP-065
    *   问题说明：PRD 5.5.1.2 规定拖拽到画布执行复制逻辑（原内容不变），但 5.5.1.5 注意事项规定单元格转节点后原单元格自动变为空白。目前用例备注已标注矛盾，需确认最终交互。
2.  **表格调整行列数缩减时的弹窗提示**：
    *   涉及用例：TP-024, REVIEW-008
    *   问题说明：PRD 未明确缩减范围时是否有弹窗提示，审核意见为“暂定”，需产品确认交互细节。
3.  **UI 视觉规范具体数值**：
    *   涉及用例：TP-033（Hover 颜色）, TP-034（预览线样式）, TP-035（遮罩颜色）
    *   问题说明：PRD 未明确具体色值和样式，审核意见为“暂定”，需 UI 设计提供具体规范。
4.  **导出功能入口及格式**：
    *   涉及用例：TP-039, TP-049
    *   问题说明：PRD 未明确导出功能的具体入口位置及支持的文件格式，审核意见为“暂定”，需产品确认。
5.  **音频播放支持**：
    *   涉及用例：TP-066
    *   问题说明：PRD 概述仅明确支持文本/图片/视频，未明确提及音频独立输入支持，需确认栅格节点是否支持音频播放。
